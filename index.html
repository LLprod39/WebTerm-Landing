Что ещё есть, если копнуть

A
Панель
User / admin dashboard: серверы, алерты, запуски Studio и агентов, быстрые переходы. Виджеты расширяются плагинами.
B
Чат-оркестратор
Диалог с ассистентом в контексте платформы — когда быстрее спросить, чем собирать граф руками.
C
Мониторинг
Health checks, алерты, watcher drafts, operational history по серверам — рядом с инвентарём, не отдельным Zabbix-клоном.
D
Безопасность runtime
Execution policy, sandbox rules, egress redaction, review gates, runtime limits. AI не «бог в root» по умолчанию.
E
Стек
Django + Channels · React/Vite/TS · Postgres · Redis · Celery · Docker Compose · nginx. MCP, LLM-провайдеры, Telegram, Keycloak — по необходимости.

DevOps / SRE

Один вход к хостам и куберу. Автоматизация в Studio. Меньше прыжков между тулзами.

Админы / NOC

Инвентарь, терминал, алерты, права. Можно не раздавать raw SSH всем подряд.

Команды с AI в ops

Агенты и MARS с логом и политиками — когда LLM уже трогают прод.

// 03 install

Скопировал. Вставил. Поднял.

Репозиторий github.com/LLprod39/WebTerm · вместо YOUR_HOST — IP или DNS

install-server.sh — Docker при необходимости, clone, env, production stack, admin.

bash 
curl -fsSL https://raw.githubusercontent.com/LLprod39/WebTerm/test/install-server.sh -o install-server.sh
chmod +x install-server.sh
./install-server.sh --repo https://github.com/LLprod39/WebTerm.git --dir /opt/webtrerm --host YOUR_HOST

пример: --host 10.0.0.15 · https: --https

Lab / local full stack.

bash 
git clone https://github.com/LLprod39/WebTerm.git
cd WebTerm
cp .env.example .env
docker compose up -d --build

health: curl http://127.0.0.1:8080/api/health/

Backend и frontend раздельно.

backend 
git clone https://github.com/LLprod39/WebTerm.git && cd WebTerm
python -m venv .venv && source .venv/bin/activate
pip install -r requirements-mini.txt
python manage.py migrate && python manage.py createsuperuser
python manage.py runserver
frontend 
cd frontend && npm ci && npm run dev

Проект только разгоняется

Модулей уже много: серверы, терминал, Studio, агенты, Kubernetes, MARS, плагины, доступы. Часть сценариев крепче, часть ещё сырая. API и UI будут ломаться и расти — это early stage, не enterprise brochure.

Если вы админ или DevOps и вам откликается идея одной on-prem консоли — клонируйте, ставьте, смотрите код, кидайте issues. Маркетинга тут нет. Есть репозиторий.
