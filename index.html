Что ещё внутри

A
Панель
Дашборд пользователя и администратора: серверы, алерты, запуски Studio и агентов, быстрые переходы. Виджеты расширяются плагинами.
B
Чат-оркестратор
Диалог с ассистентом в контексте платформы — когда быстрее спросить, чем собирать граф руками.
C
Мониторинг
Health-чеки, алерты, черновики watcher'ов, операционная история по серверам — встроено рядом с инвентарём.
D
Безопасность runtime
Execution policy, sandbox-правила, маскирование чувствительных данных, review-гейты, runtime-лимиты. AI по умолчанию работает с ограниченными правами, а не с root-доступом.
E
Стек
Django + Channels · React/Vite/TS · Postgres · Redis · Celery · Docker Compose · nginx. MCP, LLM-провайдеры, Telegram, Keycloak — по необходимости.

DevOps / SRE

Одна точка входа к хостам и Kubernetes. Автоматизация в Studio. Меньше переключений между инструментами.

Админы / NOC

Инвентарь, терминал, алерты, права. Доступ к серверам через платформу — без раздачи прямого SSH каждому.

Команды с AI в ops

Агенты и MARS с логами и политиками — контроль над тем, что AI делает в вашей инфраструктуре.

Self-hosted

Всё разворачивается у вас: Docker Compose или install-скрипт. Никакого обязательного облака.

// 03 установка

Три команды до запуска

Репозиторий github.com/LLprod39/WebTerm · вместо YOUR_HOST — IP или DNS

install-server.sh сам ставит Docker (если его нет), клонирует репозиторий, настраивает окружение и поднимает production-стек с учётной записью администратора.

bash 
curl -fsSL https://raw.githubusercontent.com/LLprod39/WebTerm/test/install-server.sh -o install-server.sh
chmod +x install-server.sh
./install-server.sh --repo https://github.com/LLprod39/WebTerm.git --dir /opt/webterm --host YOUR_HOST

пример: --host 10.0.0.15 · https: --https

Полный стек локально — для теста или лабораторного стенда.

bash 
git clone https://github.com/LLprod39/WebTerm.git
cd WebTerm
cp .env.example .env
docker compose up -d --build

health: curl http://127.0.0.1:8080/api/health/

Backend и frontend раздельно — вариант для разработки.

backend 
git clone https://github.com/LLprod39/WebTerm.git && cd WebTerm
python -m venv .venv && source .venv/bin/activate
pip install -r requirements-mini.txt
python manage.py migrate && python manage.py createsuperuser
python manage.py runserver
frontend 
cd frontend && npm ci && npm run dev

Проект в активной разработке

Модулей уже много: серверы, терминал, Studio, агенты, Kubernetes, MARS, плагины, доступы. Часть сценариев уже стабильна, часть ещё сырая. API и интерфейс будут меняться — это ранняя стадия, и мы говорим об этом честно.

Если вы админ или DevOps и вам близка идея одной self-hosted консоли — клонируйте репозиторий, разверните у себя и посмотрите сами. Баги и предложения — в issues на GitHub.
